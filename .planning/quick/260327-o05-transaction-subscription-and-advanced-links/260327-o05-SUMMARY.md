# Quick Task 260327-o05: Transaction Subscription and Linked Setup — Summary

**One-liner:** El registro manual de transacciones ahora permite marcar suscripciones y expandir un bloque liviano para crear destinatarios o pagos recurrentes desde el mismo submit.

## What Changed

- `transactionSchema` ahora incluye `is_subscription` y un schema auxiliar para las opciones avanzadas del create flow.
- `createTransaction()` prepara destinatario y/o plantilla recurrente antes de guardar la transacción, y limpia esos registros si la inserción final falla.
- `TransactionForm` agrega un switch visible para suscripciones y un bloque expandible con creación opcional de destinatario y pago recurrente.
- `MobileTransactionForm` replica el mismo comportamiento, con guardas para no ofrecer esas opciones en transferencias.
