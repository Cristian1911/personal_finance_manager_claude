2. `showStep(n)`:
   - Hide all step divs (display: none)
   - Show step n div (display: block), add `step-enter` class for animation
   - Remove `step-enter` class after 200ms (so re-entry re-triggers animation)
   - Update progress bar fill width: `(n / 6) * 100 + '%'`
   - Update step counter text: `'Paso ' + n + ' de 6'`
   - Update `aria-valuenow` on progress bar
   - If n === 6: hide progress section, show scorecard section
   - Else: show progress section, hide scorecard section
3. `nextStep()`: increment currentStep (max 5 for steps 1-4, step 5 uses simulateSubmit)
4. `prevStep()`: decrement currentStep (min 1)
5. Purpose selection: click handler on purpose buttons — toggle selected class, store selected purpose, enable/disable Next button
6. `simulateSubmit()` (step 5 Finalizar):
   - Disable button, change text to spinner SVG + "Guardando..."
   - `setTimeout(800)` then call `showStep(6)`
7. `cycleTab(position)` (step 4):
   - TAB_OPTIONS array with 6 entries matching page.tsx: `[{id:'transactions', label:'Movimientos', icon:'ArrowLeftRight'}, {id:'presupuesto', label:'Presupuesto', icon:'PiggyBank'}, {id:'debt-recurring', label:'Deudas', icon:'CreditCard'}, {id:'categories', label:'Categorias', icon:'Tag'}, {id:'destinatarios', label:'Destinatarios', icon:'Users'}, {id:'import', label:'Importar', icon:'Upload'}]`
   - Track tab2Index and tab3Index
   - On cycle: increment index, skip if same ID as the other position's current tab (mirror page.tsx swapTab dedup logic)
   - Update the tab button's icon and label text in the DOM
