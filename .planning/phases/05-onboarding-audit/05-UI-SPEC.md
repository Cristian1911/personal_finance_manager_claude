## Spacing Scale

Declared values from `globals.css` `--z-space-*` tokens. HTML file must replicate using inline styles or `<style>` block.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline padding |
| sm | 8px | Compact element spacing, label-to-input gap |
| md | 16px | Default element spacing, card content padding |
| lg | 20px | Card section breaks, between form groups |
| xl | 28px | Card header padding |
| 2xl | 40px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Project Token Inheritance:** `lg` (20px), `xl` (28px), and `2xl` (40px) are not in the standard 8-point set {4, 8, 16, 24, 32, 48, 64}. These values are inherited directly from Zeta's custom `--z-space-*` token scale defined in `webapp/src/app/globals.css`. The HTML artifact inherits this scale by design to preserve visual fidelity with the source application. These are project-specific exceptions, not deviations from the spec.

Exceptions:
- Progress bar height: 8px (fixed — matches `h-2` in source)
- Card footer padding: 24px top/bottom — from `p-6` (24px) in source `CardFooter`
- Touch targets on purpose-selection buttons: minimum 56px height — large tap area matching `p-4` + icon + text
- Step counter badge max-width: `max-w-lg` (512px) — matches source `max-w-lg` container

## Step-by-Step Interaction Contract

### Wizard navigation
- One step visible at a time. JavaScript manages a `currentStep` variable (1–6).
- Back button: `currentStep > 1 ? currentStep-- : disabled`. Step 1 back button is disabled.
- Next button on steps 1–4: advances `currentStep++`. No async call.
- "Finalizar" button on step 5: simulates submit (no real Supabase call), then advances to step 6.
- Progress bar updates on every step change via `style.width` calculation.
- Step transition: CSS `opacity: 0 -> 1` + `transform: translateX(16px) -> translateX(0)` over 200ms. Replicate `animate-in fade-in slide-in-from-right-4 duration-200`.

### Step 1 — Objetivo
- 4 purpose cards rendered in a grid (single column).
- Click selects a purpose; re-click does NOT deselect (radio-button semantics).
- Next button disabled until a purpose is selected.
- Pre-filled state: `manage_debt` selected (from CONTEXT.md D-07).

### Step 2 — Tu perfil
- Full name text input + currency select.
- Pre-filled: name = "María García", currency = "COP".
- Next button disabled until name field is non-empty.

### Step 3 — Pulso mensual
- Income + expenses CurrencyInput fields.
- Conditional: "disponible para presupuesto" hint div shown when income > 0 (static in HTML — always shown with pre-filled values).
- Conditional: debt count input shown only when purpose === `manage_debt` (always shown in pre-filled state).
- Debt count pre-filled: 2.
- Confirmation text below debt count shown when value > 0 (always shown).
- Pre-filled: income = $4,500,000 / expenses = $3,200,000 / debtCount = 2.

### Step 4 — Tu app (tab preview)
- Phone mockup with 4 tabs: Tab 1 "Inicio" (fixed, active), Tab 2 (tappable), Tab 3 (tappable), Tab 4 "Más" (fixed).
- Pre-filled for `manage_debt` purpose: Tab 2 = "Deudas" (CreditCard icon), Tab 3 = "Presupuesto" (PiggyBank icon).
- Tapping Tab 2 or Tab 3 cycles through available tab options (pure JS swap, no server call).
- Phone mockup inner screen area: placeholder "Tu dashboard" centered text.

### Step 5 — Primera cuenta
- Account name text input, account type select, balance CurrencyInput.
- Pre-filled: name = "Bancolombia Ahorros", type = SAVINGS, balance = $1,250,000.
- "Finalizar" button disabled until name and balance are non-empty.
- Loading state on Finalizar: spinner icon + disabled state for 800ms simulated delay, then advance to step 6.

### Step 6 — Completion (Quick win)
- No progress bar shown (step 6 is post-wizard).
- Success icon centered, personalized greeting: "Listo, María!".
- Primary CTA: "Importa tu primer extracto" (links to `#` in HTML artifact).
- Secondary CTA: "Explorar primero" (links to `#`).

## Annotation Callout Spec

Annotations appear below each step card, separated by 16px gap. They are never overlaid.

**Annotation box structure:**
```html
<div class="annotation-box">
  <div class="annotation-item concern">⚠️ <strong>Concern:</strong> description</div>
  <div class="annotation-item positive">✅ <strong>Good pattern:</strong> description</div>
</div>
```

**Annotation box styles:**
- Background: `color-mix(in srgb, #C5BFAE 6%, transparent)` (very subtle sage tint)
- Border: `1px solid rgba(197, 191, 174, 0.12)`
- Border radius: 12px
- Padding: 16px
- Font size: 15px / weight 400 / line-height 1.6

**Annotation item styles:**
- `.concern`: icon color `#E8875A` (z-expense), text color `#D8D2C4` (z-sage-light)
- `.positive`: icon color `#5CB88A` (z-income), text color `#D8D2C4` (z-sage-light)
- Spacing between items: 8px

**Required annotation topics per step (exact copy determined by executor after auditing source code):**

| Step | Expected annotation topics |
|------|---------------------------|
| Step 1 | Purpose selection validation, no skip option, purpose influences step 3 and step 4 conditionals |
| Step 2 | Currency defaults to USD (not COP), no timezone displayed to user, name required but surname not captured |
| Step 3 | Income/expenses required but debt count optional for manage_debt, "disponible" hint is additive/positive, debtCount not validated against account types |
| Step 4 | Tab swap is ephemeral until finishOnboarding() fires, no explanation of what each tab feature does, phone mockup body is placeholder |
| Step 5 | No multi-account entry in onboarding, account type defaults to CHECKING, balance can be 0 but button requires non-empty string |
| Step 6 | Quick win is purpose-aware, secondary CTA goes to /dashboard which may show empty state without data |

## Implementation Constraints (Phase 5 specific)

These override general defaults where the static HTML context differs from a React/Next.js page.

| Constraint | Value |
|-----------|-------|
| Output file | `ui-showcases/onboarding-walkthrough.html` |
| External dependencies | Tailwind CDN (`cdn.tailwindcss.com`), Lucide icons via unpkg CDN |
| JavaScript | Vanilla JS only — `<script>` block in the HTML file. No ES modules, no bundler. |
| Supabase calls | None — all form submissions are simulated |
| React/JSX | None — pure HTML |
| Server required | None — file opens directly in browser (`file://` protocol) |
| Minimum browser | Modern Chromium (Chrome 120+) — no IE/legacy support needed |
| Viewport | Must render correctly at 375px (mobile) and 1280px (desktop) |
| Accessibility | `aria-label` on icon-only buttons, `role="progressbar"` on progress bar, `aria-current="step"` on active step indicator |
