## Spacing Scale

Declared values (all multiples of 4, with project exceptions noted):

| Token | CSS Variable | Value | Usage | Justification |
|-------|-------------|-------|-------|---------------|
| xs | `--z-space-xs` | 4px | Icon gaps, inline chip padding | Base of 4-point scale |
| sm | `--z-space-sm` | 8px | Compact row padding, badge inner padding | 2x base |
| md | `--z-space-md` | 16px | Default card padding, form element spacing | 4x base |
| lg | `--z-space-lg` | 20px | Section internal padding | 5x base; bridges card padding and section gaps without jumping to 24px |
| xl | `--z-space-xl` | 28px | Between sections inside the same tier | 7x base; visually separates sections without the density of 24px or the airiness of 32px |
| 2xl | `--z-space-2xl` | 40px | Between dashboard tiers (hero → health score) | 10x base; creates clear tier-level breathing room between primary sections |
| 3xl | `--z-space-3xl` | 64px | Page-level top/bottom padding | 16x base |

**Phase-specific exceptions:**
- Hero "Disponible para gastar" number: `py-6` (24px) vertical clearance above and below
- Speedometer gauge container: minimum height 160px on mobile, 200px on desktop
- Health meter rows inside the speedometer section: `px-3 py-2.5` — grandfathered from existing `MeterRow` component; do not replicate in new components, use `py-2` (8px) or `py-3` (12px) instead
- Touch targets for interactive meter rows: minimum 44px height (accessibility floor)
- Debt-free banner: `px-4 py-3` (16px / 12px) — compact but readable

## Typography

Four roles, two weights. All sizes reference the `--z-type-*` token system.

| Role | CSS Token | Size | Weight | Line Height | Tracking | Usage |
|------|-----------|------|--------|-------------|----------|-------|
| Display | `--z-type-display-*` | 52px | 700 | 1.0 | -0.04em | Hero "Disponible para gastar" number |
| Heading | `--z-type-h1-*` | 28px | 700 | 1.2 | -0.02em | Health score number in speedometer; section titles |
| Body | `--z-type-body-*` | 15px | 400 | 1.7 | 0 | Status headlines, section subtitles, narrative copy |
| Label | `--z-type-label-*` | 11px | 400 | 1.4 | 0 | Section header text (uppercase + tracking-wide), KPI sub-labels |

**Declared weights:** 700 (semibold) and 400 (regular) only. The 52px Display size provides sufficient hierarchy over 28px Heading without a separate weight graduation.

**Additional widget sizes (already established in `kpi-widget.tsx`):**
- Widget large value: 22px / weight 700 — KPI card primary numbers
- Widget caption / trend: 11px / weight 400 — KPI secondary text

**Rule:** Only these four roles (+ widget variant) may be used. No one-off `text-[Npx]` outside of these established sizes.

## Color

The Zeta token layer defines the full palette. This section maps the 60/30/10 split to existing tokens and reserves accent usage for this phase.

| Role | Token | Resolved Value | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | `--z-ink` | #0A0E14 | Page background (`--background`) |
| Secondary (30%) | `--z-surface` / `--z-surface-2` | #131720 / #1C1F28 | Cards, section backgrounds, hero tint |
| Primary tier bg | `--z-surface-2` | #1C1F28 | Hero + health score "floating" sections (no card border) |
| Secondary tier bg | `--z-surface` | #131720 | All other dashboard cards (`bg-card`) |
| Accent (10%) | `--z-sage` | #C5BFAE | Section toggle controls, inactive state UI chrome |
| Destructive | `--z-debt` | #E05545 | `--destructive` mapping; over-budget headline indicator |

**Financial semantic colors — reserved for specific elements only:**

| Token | Value | Reserved For |
|-------|-------|-------------|
| `--z-income` | #5CB88A | Under-budget headline dot, positive trend arrows, savings/income meter tags |
| `--z-expense` | #E8875A | Near-limit headline dot, expense meter tags, `alto` level indicators |
| `--z-debt` | #E05545 | Over-budget headline dot, debt meter tags, `critico` level indicators, destructive actions |
| `--z-alert` | #D4A843 | `atento` level indicators, near-limit zone on speedometer |
| `--z-excellent` | #3D9E6E | `excelente` level indicator, best zone on speedometer |

**Speedometer gauge color zones (left-to-right, 0-100 score):**
- 0–25: `--z-debt` (red)
- 25–50: `--z-expense` + `--z-alert` gradient (orange → amber)
- 50–75: `--z-income` (green)
- 75–100: `--z-excellent` (deep green)

**Information tier visual distinction:**
- Primary tier (hero + health score): no Card wrapper, no border, `bg-z-surface-2` background tint, larger vertical padding (`py-6`)
- Secondary tier (all other sections): standard `Card` wrapper with `border-z-border`, normal padding (`py-4`)

**Debt-free banner color:** `surface-income` utility class (8% opacity green background, green text + border) when date is in the future. Not an accent element — it uses the `--z-income` semantic token.

## Interaction Contracts

### Status Headline (DASH-01)
- Renders as a single sentence below the 3 KPI cards in the hero section
- Colored dot (8px circle) precedes the text, color matches budget status
- No click target — display only
- Streams in with tier 1 (hero) — no additional data fetch needed (budget pace derived from hero data)
- When no budget/transaction data: renders prompt-to-action "Importa tu extracto para ver tu estado" in `text-muted-foreground` (no colored dot)

### Debt-Free Banner (DASH-04)
- Renders below the status headline inside the hero container
- When no active debts: component returns `null` — no placeholder, no empty space
- When debts exist: colored strip, `surface-income` utility class, format: "Libre de deudas: Dic 2028 (32 meses)"
- Streams with tier 2 data (debt countdown is already tier 2)
- Banner skeleton: single 40px-tall rounded rectangle at full width

### Health Score Section (DASH-02 + DASH-03)
- Positioned immediately after hero, before cash flow
- Primary tier visual treatment: no Card border, `bg-z-surface-2` background, `py-6` padding
- Speedometer renders on client only (lazy via `next/dynamic` with `ssr: false`)
- Score number: 28px / weight 700, centered inside gauge arc
- Score label ("Bien", "Excelente"): 11px / weight 400 below the number
- Individual horizontal meters below gauge — reuses existing `MeterRow` component
- Meter click behavior unchanged: opens `HealthMeterExpanded` sheet

### Section Subtitles (DASH-05)
- Each `DashboardSection` receives a computed `subtitle` string from its parent
- Subtitle renders as 15px / weight 400 / `text-muted-foreground` on a single line below the section header toggle button
- When subtitle is absent (no data), section header renders without subtitle line — no empty space
- Subtitle is a Server Component computation — no client-side state needed

### Dashboard Section Order (DASH-03)
Fixed order, no drag/drop:
1. Hero (primary tier, no section wrapper)
2. Health Score (primary tier, no section wrapper)
3. Cash Flow (`DashboardSection` with subtitle)
4. Budget (`DashboardSection` with subtitle)
5. Activity Heatmap (`DashboardSection` with subtitle)
6. Accounts (`DashboardSection` with subtitle)
7. Debt Progress (`DashboardSection` with subtitle)
8. Net Worth (`DashboardSection` with subtitle)

## Copywriting Contract

All strings are Spanish. Tone: friendly coach — second person, encouraging but honest. Never: "MAL", "PELIGRO", "FALLASTE", "TERRIBLE", guilt-inducing absolute statements.

### Status Headlines (DASH-01)

| Budget State | Threshold | Copy | Dot Color |
|-------------|-----------|------|-----------|
| Under budget | spent < 90% of budgeted | "Vas bien este mes — gastos dentro del presupuesto" | `--z-income` |
| Near limit | spent 90–100% of budgeted | "Cerca del límite — queda poco margen este mes" | `--z-alert` |
| Over budget | spent > 100% of budgeted | "Te pasaste del presupuesto — $X por encima" | `--z-debt` |
| No data | no budget or no transactions | "Importa tu extracto para ver tu estado" | none (muted) |

### Debt-Free Banner (DASH-04)

| State | Copy |
|-------|------|
| Active debts | "Libre de deudas: {Mes Año} ({N} meses)" |
| No debts | (component hidden — no copy) |

### Health Score Labels (DASH-02)

| Score Range | Label |
|-------------|-------|
| 0–25 | "Crítico" |
| 26–50 | "Atento" |
| 51–75 | "Bien" |
| 76–90 | "Sólido" |
| 91–100 | "Excelente" |

### Section Subtitles (DASH-05)

| Section | With Data | No Data |
|---------|-----------|---------|
| Flujo de Caja | "Gastaste $X de $Y este mes — {assessment}" | "Importa tu extracto para ver tu flujo" |
| Presupuesto | "Necesidades al {N}% — {assessment}" | "Configura tu presupuesto en ajustes" |
| Cuentas | "{N} cuentas activas — saldo total {$X}" | "Agrega una cuenta para comenzar" |
| Deudas | "{N} deudas activas — {$X} pagado este mes" | "Sin deudas registradas — ¡excelente!" |
| Patrimonio | "Patrimonio neto: {$X}" | "Agrega cuentas para calcular tu patrimonio" |
| Heatmap | "Actividad de los últimos {N} meses" | "Sin transacciones registradas" |

Assessment words (inline in subtitle, match budget state):
- positive: "vas bien"
- neutral: "dentro del presupuesto"
- caution: "cerca del límite"
- critical: "por encima del presupuesto"

### Empty States

| Context | Heading | Body | CTA |
|---------|---------|------|-----|
| No budget configured | "Sin presupuesto" | "Configura tu presupuesto para ver tu estado mensual" | "Ir a ajustes" → `/settings` |
| No transactions this month | "Sin movimientos este mes" | "Importa tu extracto bancario para ver tus gastos" | "Importar extracto" → `/transactions/import` |
| No debts | — | (DebtFreeBanner hidden; no empty state shown — clean hero) | — |
| No income configured | "Sin datos de ingresos" | "Agrega tu ingreso mensual para activar los indicadores de salud" | "Configurar ingresos" → `/settings` |

### Error States

| Context | Copy |
|---------|------|
| Health score computation fails | "No se pudo calcular tu puntaje — intenta recargar la página" |
| Debt countdown fetch fails | (banner hidden silently — no error shown to user) |
| Hero data fetch fails | (handled by existing `DashboardError` boundary at layout level) |

### Destructive Actions

No destructive actions in this phase. Dashboard redesign is display-only — no deletions, no confirmations required.

## Animation Contract

Phase 4 established the CSS animation pattern. This phase extends it:

| Trigger | Animation | Classes |
|---------|-----------|---------|
| Health score section streams in | Fade + slide from bottom | `animate-in fade-in slide-in-from-bottom-4 duration-200` |
| Debt-free banner appears | Fade in | `animate-in fade-in duration-150` |
| Status headline | No animation — tier 1, synchronous | — |
| Speedometer needle (on mount) | CSS transition from 0 to computed angle | `transition-transform duration-500 ease-out` |

No Framer Motion. No `AnimatePresence`. Enter-only animations. No exit animations.

## Accessibility

- Status headline colored dot must have `aria-label` describing the budget state
- Speedometer gauge: `role="meter"` + `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`, `aria-label="Puntaje de salud financiera"`
- Health meter rows retain existing keyboard focus ring: `focus-visible:ring-[3px] focus-visible:ring-ring/50`
- Debt-free banner: no interactive role needed — display only
- Section subtitle: rendered as `<p>` below the section `<button>` header, not inside it
