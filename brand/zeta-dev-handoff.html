01 — Design Tokens

Color System

Zeta uses a 60-30-10 rule: Ink Black (60%) for backgrounds, Sage Beige (30%) for text and surfaces, functional accents (10%) for financial data. Colors are semantically tied to financial meaning.

Core Palette

TokenValueRoleUsage %
--z-ink #1C1C1C Base background 60%
--z-sage #C5BFAE Primary text, surfaces 30%
--z-sage-light #D8D2C4 Light variant for emphasis —
--z-sage-dark #9E9888 Muted text, captions —
--z-white #F5F3EF Primary text on dark backgrounds —

Functional Accents (Financial Semantics)

Critical rule: These colors carry meaning. Green = money coming in. Amber = money going out. Red = obligation/debt. Gold = attention needed. Never use them decoratively.

06 — Badge Color Matrix

Status Badge Styles

Each badge state maps to a financial meaning. Pattern: text = semantic color, border = ~20% tint, bg = ~8% tint.

/* Financial status badges */
.badge-pending  { color: #E9C46A; border-color: #4a3e18; background: #231e0c; } /* ⏳ Pending    — alert gold   */
.badge-progress { color: #F4A261; border-color: #6b3e15; background: #2d1f0e; } /* ◌ In progress — expense amber */
.badge-success  { color: #52B788; border-color: #1a4a30; background: #0e2419; } /* ✓ Completed  — income green  */
.badge-failed   { color: #C44536; border-color: #5c1a14; background: #2e0d0a; } /* ✗ Failed     — debt red      */
.badge-expired  { color: #888888; border-color: #444444; background: #222222; } /* ○ Expired    — neutral gray  */
Semantic mapping: badge-success = income green (money received). badge-failed = debt red (obligation missed). badge-progress = expense amber (processing outflow). badge-pending = alert gold (needs attention).
07 — States & Interactions

Component States

Interaction patterns for common states across Zeta's UI.

Loading States
While fetching financial data
KPI values: skeleton pulse animation on value area (surface-2 shimmer).
Timeline: placeholder rows with 3 skeleton items.
Bubbles: collapsed to uniform small circles, gray.
Duration: skeleton shimmer — 1.5s ease-in-out infinite.
Empty States
No data available
Show the Z motif at 0.05 opacity centered, with a single line of sage-colored text below.
Copy examples: "Aún no hay transacciones este mes", "Agrega tu primera cuenta para empezar."
Never show raw "No data" — always guide next action.
Error States
Failed to load or sync
Use --z-debt red for error indicator dot (8px).
Message in sage-dark: "No pudimos sincronizar. Intenta de nuevo."
Retry button: ghost style, sage border, 20px radius pill.
Hover / Tap
Interactive feedback
Cards: subtle brightness increase — filter: brightness(1.05), 150ms ease.
Badges: no hover state (informational only).
Timeline items: background-color transitions to surface-2, 150ms ease.
Bubbles: scale(1.08) with 200ms ease on hover/tap.
08 — Accessibility

A11y Notes

Color Contrast
--z-sage (#C5BFAE) on --z-ink (#1C1C1C): contrast ratio ~9.2:1 (AAA).
--z-income (#52B788) on --z-ink: ~7.8:1 (AAA).
--z-expense (#F4A261) on --z-ink: ~8.4:1 (AAA).
--z-debt (#C44536) on --z-ink: ~4.5:1 (AA).
--z-alert (#E9C46A) on --z-ink: ~9.1:1 (AAA).
All pass AA minimum. Debt red is the tightest — don't reduce its size below 14px.
Screen Reader & ARIA
KPI widgets: role="status" + aria-label with full sentence ("Saldo disponible: $4,280, subió 12.4%").
Trend arrows (↑↓): use aria-hidden="true", meaning conveyed by aria-label on parent.
Badges: role="status".
Bubbles: role="img" + aria-label describing proportions.
Timeline: <ol role="list"> with <li> for each item.
09 — Ecosystem

Sister Brand: Venti5 Labs

Shared: Inter font, dark-mode-first, surface/border system. Zeta uses financial-semantic colors; Venti5 uses per-service palettes. When co-existing, each brand keeps its own accent colors within its own UI zone.

Cross-brand rule: If Zeta and Venti5 content appear together (e.g. a dashboard linking to both), use neutral darks and let each brand's accent colors speak for their own elements only. Never mix Zeta's income green with Venti5's service green — they serve different semantic purposes.
10 — Status Surfaces

Status Surface Pattern

The badge color matrix (section 06) applies at any scale — not just small badges, but also cards, callouts, sections, and timeline groups. This is Zeta's primary tool for giving visual meaning to containers.

The formula: text = semantic color · border = semantic color at 20% opacity · background = semantic color at 8% opacity. This creates a tinted surface that conveys meaning without competing with content.

Surface Definitions

/* Status surfaces — use these for cards, callouts, timeline groups, sections */

/* Income / Success / Positive */
.surface-income  { color: var(--z-income);  border-color: color-mix(in srgb, var(--z-income) 20%, transparent);  background: color-mix(in srgb, var(--z-income) 8%, transparent); }
/* Tailwind: text-z-income border-z-income/20 bg-z-income/[0.08] */

/* Expense / In Progress / Warning */
.surface-expense { color: var(--z-expense); border-color: color-mix(in srgb, var(--z-expense) 20%, transparent); background: color-mix(in srgb, var(--z-expense) 8%, transparent); }
/* Tailwind: text-z-expense border-z-expense/20 bg-z-expense/[0.08] */

/* Debt / Failed / Danger */
.surface-debt    { color: var(--z-debt);    border-color: color-mix(in srgb, var(--z-debt) 20%, transparent);    background: color-mix(in srgb, var(--z-debt) 8%, transparent); }
/* Tailwind: text-z-debt border-z-debt/20 bg-z-debt/[0.08] */

/* Alert / Pending / Attention */
.surface-alert   { color: var(--z-alert);   border-color: color-mix(in srgb, var(--z-alert) 20%, transparent);   background: color-mix(in srgb, var(--z-alert) 8%, transparent); }
/* Tailwind: text-z-alert border-z-alert/20 bg-z-alert/[0.08] */

/* Neutral — for future/inactive states. Clean, no tint. */
.surface-neutral { color: var(--z-sage-dark); border-color: var(--z-border); background: var(--z-ink); }
/* Tailwind: text-muted-foreground border-border bg-background */

When to Use

Timeline Date Groups
Past/overdue dates → surface-debt
Today's date → surface-alert
Future dates → surface-neutral
Insight/CTA Cards
Savings opportunity → surface-income
Spending alert → surface-expense
Debt warning → surface-debt
Informational tip → surface-alert
Balance/Amount Colors
Positive balance/inflow → text-z-income
Expense amount → text-z-expense
Debt/negative amount → text-z-debt
Interest costs → text-z-expense
Anti-pattern: Never use bg-muted/30 or bg-primary/5 for status-carrying containers. These produce muddy, low-contrast surfaces on dark backgrounds. Use the status surface pattern instead — it guarantees contrast and semantic clarity.

Gradient Fills

Area charts use vertical linear gradients with the semantic color:
Top stop: semantic color at 40% opacity.
Bottom stop: semantic color at 5% opacity.
This ensures visibility on dark backgrounds without overwhelming the chart.
