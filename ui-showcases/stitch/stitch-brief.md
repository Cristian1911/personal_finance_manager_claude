# Stitch Brief — Zeta Refinement

Paste this into Stitch after uploading `current-state-board.png`. If needed, also upload one or more files from `inputs/`.

## Prompt

Design a refined UI direction for Zeta, a personal finance app for Colombian users.

Use the uploaded screenshots as the current product state. Keep the app grounded, operational, and trustworthy. Do not turn it into a generic glossy fintech landing page. Preserve the sober dark theme, Spanish-first language, and the feeling that this is a real money tool, not a marketing mockup.

The main problem to solve is this: the product has a lot of capability, but the experience lacks a clear direction. It currently feels like a map of internal modules instead of one guided operating model.

The new direction should be:

- one dominant story: "¿Voy bien y qué hago ahora?"
- lower anxiety
- clearer hierarchy
- fewer first-class nouns
- more guided daily use

Reorganize the information architecture around four top-level areas:

- Inicio
- Movimientos
- Plan
- Más

Use these product rules:

- Inicio should answer status first, then show the next 2 to 3 actions with highest impact.
- Movimientos should combine import, quick capture, review, and transaction cleanup into one coherent working flow.
- Plan should connect budget, debt, recurring payments, and obligations under one lens: what pressure is coming and what decision improves the month.
- Más should hold admin and setup surfaces such as cuentas, categorías, destinatarios, analytics, and settings.

Design principles:

- Spanish-first UI
- strong hierarchy, not card sprawl
- one hero area per page
- fewer equally weighted modules
- preserve functional density, but reduce visual noise
- optimize for both desktop and mobile
- use labels and structure that feel practical, not startup-generic
- do not add random gradients, empty marketing illustrations, or trendy fintech fluff

Please generate:

1. a refined desktop direction for Inicio, Movimientos, Plan, and Más
2. a matching mobile direction for the same four areas
3. a clearer navigation model
4. stronger prioritization of "Disponible para gastar", urgent obligations, and next actions
5. a visual system that feels more intentional and less like a collection of separate tools

If helpful, simplify or merge modules, but keep the app believable as a real production finance tool.

## Short version

Refine this finance app so it feels like a guided command center instead of a broad admin system. Keep it dark, serious, Spanish-first, and practical. Reduce top-level complexity. Make Inicio answer "¿Voy bien?" immediately, then drive the next action.
