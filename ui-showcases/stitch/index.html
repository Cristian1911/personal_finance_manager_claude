Zeta / Stitch handoff

Current state, curated for redesign.

The raw showcases are good archives, but weak redesign prompts. This package shrinks the app down to the screens and principles that should guide the next refinement pass.

Direction: Estado y siguiente paso Top-level IA: Inicio / Movimientos / Plan / Más Goal: answer “¿Voy bien?” fast
What exists Strong real flows: dashboard, transactions, import, debt.
Main problem Too many first-class nouns and too much equal-weight UI.
Why redesign The product reads like a system map instead of an operating model.

Use This In Stitch

  • Upload current-state-board.png first.
  • Add any file from inputs/ for extra detail on a single flow.
  • Paste the prompt from stitch-brief.md.
  • Keep the direction from ../ship-now/01-estado-y-siguiente-paso.html.
Stitch officially supports image inputs such as screenshots or wireframes, and can later move the result into Figma.

Key Review Points

  • Dashboard needs a single dominant status story.
  • Transactions should feel like an inbox flow, not a toolbox.
  • Debt, budget, and recurring should read as one planning system.
  • Manage belongs under Más, not as a peer to core daily work.
